数据结构学什么 ​

1.如何用程序代码把现实世界的问题信息化

2.如何用计算机高效的处理这些信息而创造价值

基本概念 ​

数据 ​

数据是信息的载体，描述客观事物属性的数字，字符或者是所有能输入到计算机中能被计算机

识别或处理的符号的集合，是程序加工的原料

  • 早期的计算机处理的数据-纯数值型问题
  • 现代的计算机处理的数据-经常处理非数值的问题

数据元素>>数据项 ​

数据元素用于描述一个个个体

数据元素是数据的基本单位，通常作为一个整体进行考虑和处理

一个数据元素可以由若干个数据项组成，数据项是构成数据元素的不可分割的最小单位

数据对象 ​

数据对象是具有相同性质的数据元素的集合，是数据的一个子集

数据结构 ​

数据结构是数据元素相互之间存在一种或多种特定关系的数据元素的集合

同样的数据元素可以组成不同的数据结构

不同的数据元素可以组成同样的数据结构

  • 线性数据结构–排行榜
  • 网状数据结构–朋友圈

数据类型、抽象数据类型 ​

数据类型是一个值集合（值的取值服务，如bool型的值范围为true、false）和定义在此集合上的一组操作（如int的加减乘除）的总称。

  1. 原子类型。其值不可再分的数据类型
  2. 结构类型。其值可以再分解为若干成分（分量）的数据类型

抽象数据类型（abstract Data Type,ADT)是抽象数据组织及与之相应的操作

  • 定义一个ADT就是在“定义“一种数据结构（逻辑结构，数据的运算）
  • 确定了ADT的存储结构，才能”实现”这种数据结构（物理结构）

数据结构的三要素 ​

逻辑结构 ​

数据元素之间的逻辑关系

  1. 集合-各个元素同属于一个集合，别无其他关系
  2. 线性结构-数据元素之间是一对一的关系
    • 除了第一个元素，所有元素都有唯一前驱
    • 除了最后一个元素，所以元素都有唯一后继
  3. 树形结构
    • 数据元素之间是一对多的关系
  4. 图结构
    • 数据元素之间是多对多的关系

数据的运算 ​

针对某种逻辑结构，结合实际需求，定义基本运算

  • 运算的定义是针对逻辑结构的，指出运算的功能
  • 运行的实现是针对存储结构的，指出运算的具体操作步骤

线性结构->基本运算

  • 查找第i个数据元素
  • 在第i个位置插入新的数据元素
  • 删除第i个位置是数据元素

物理结构（存储结构） ​

如何使用计算机实习这种数据结构->如何用计算机表示数据元素的逻辑关系

顺序存储

  • 顺序存储

非顺序存储

  • 链式存储
  • 索引存储
  • 散列存储
  1. 若采用顺序存储，则各个数据元素在物理上必须是连续的；若采用非顺序存储，则各个元素在物理上可以是离散的
  2. 数据的存储结构会影响存储空间的分配的方便程度
  3. 数据的存储结构会影响对数据的运算速度

例：

顺序存储，把逻辑上相邻的元素存储在物理位置也相邻的存储单元中，元素之间的关系由存储单元的邻接关系来体现

链式存储，逻辑上相邻的元素在物理地址上可以不相邻，借助指示元素的存储地址的指针来表示元素之间的逻辑关系

索引存储，在存储元素信息的同时，建立附加的索引表。索引中的每项称为索引项，索引项的一般形式是（关键字，地址）

散列存储，根据元素的关键字直接计算出该元素的存储地址，又称哈希（Hash）存储

算法 ​

程序=数据结构+算法

算法是对特定问题求解步骤的一种描述，它是指令的有限序列，其中的每条指令表示一个或多个操作

算法的特性 ​

  1. 有穷性—>一个算法必须总在执行有穷步后结束，且每一步都可以在有穷时间内完成（算法必须是有穷的，程序可以是无穷的）
  2. 确定性 —>每条指令都必须有确切的含意，对于相同的输入只能得到相同的输出
  3. 可行性—>算法中描述的操作都可以通过已经实现的基本运算执行有限次来实现

输入。一个算法可以有零个或多个输入，这些输入取自某个特定的对象或集合

输出。一个算法有一个或多个输出，这些输出是与输入有着某种特定关系的量

如数学中函数

“好“算法的特质 ​

  1. 正确性，一个算法能正确的求解问题
  2. 可读性，具有良好的可读性，帮助人理解
  3. 健壮性，输入非法数据时，算法能适当的做出反应或进行处理，不会产生莫名结果
  4. 高效率与低存储需求（时间、空间复杂度低）

如何评估算法时间开销 ​

事后统计问题 ​

  1. 与机器性能有关
  2. 与编程语言有关，越高级的语言效率越低
  3. 与编译程序产生的机器指令质量有关
  4. 有些算法不能事后再统计

算法的时间复杂度 ​

事先评估算法时间开销T与问题规模n的关系

例：

T=3n+3

当n=3000时

3n = 9000vs3n+3 = 9003
n^2 = 9,000,000vsn^2 +3n+100 = 9,010,000
n^3=27,000,000,000vsn^3+n^2=27,018,000,000

如何计算 ​

当n的规模够大时，可以只考虑阶数高的部分

顺序执行的代码只会影响常数阶，可以忽略

只需要挑循环其中一个基本操作分析执行次数与n的关系即可

嵌套循环时只需要关注最深层代码循环的次数

T1(n)= O(n)

T2(n)=O(n^2)

T3(n)=o(n^3)

大O表示同阶，同等数量级。即当n->无穷时，二者之比为常数

常用规则 ​

  • 加法规则（多项相加，只保留最高阶的项，且系数变为1）
  • 乘法规则 （多项相乘，都保留）

时间复杂度排序(常对幂指阶)

O(1)<O(log2n)<O(nlog2n)<O(n^2)<O(n^3)<O(2^n)<O(n!)<O(n^n)

三种复杂度 ​

最坏时间复杂度：最坏情况下算法的时间复杂度

平均时间复杂度：所有输入示例等概率出现的情况下，算法的期望运行时间

最好时间复杂度：最好情况下算法的时间复杂度

  1. 找到所占空间大小与问题规模相关的变量
  2. 分析所占空间x与问题规模n的关系
  3. x的数量级O(x)就是算法空间复杂度S(n)

只需要分析数据在内存中所占的大小

计算算法的空间复杂度时，只需要关注存储空间大小与问题规模相关的变量

递归程序 ​

  1. 找到递归调用的深度x与问题规模n的关系
  2. x的数量级O(x)就是算法空间复杂度S(n)

空间复杂度=递归调用的深度

线性表 ​

定义（逻辑结构） ​

具有相同数据类型(每个数据元素所占的空间一样大)的n个数据元素的有限序列(有次序)，其中n为表长，当n=0时线性表为一个空表

如果用L命名一个线性表那么可以像如下表示

L=（a1,a2,a3,…,an）

几个概念：

  1. ai是线性表中的”第i个“元素线性表中的位序
  2. a1是表头元素，an是表尾元素
  3. 除了第一个元素外，每个元素都有且仅有一个直接前驱
  4. 除了最后一个元素，每个元素有且仅有一个直接后驱

操作（运算） ​

  • 初始化表，构造一个空的线性表L，分配内存空间

  • 销毁操作，销毁线性表，释放所占的内存空间

  • 插入操作，在表L的第i个位置上插入指定元素e

  • 删除操作，删除表L第i个位置元素

  • 按值查找，在表L中查找具有给定关键字值的元素

  • 按位查找，获取表L中第i个位置的元素的值

    其他操作

    • 求表长，返回表L的长度，即表中元素的个数
    • 输入操作，从前往后顺序输出线性表L中的每个元素值
    • 判空操作，表L为空时返回true，否则返回false

存储/物理结构 ​

顺序表（顺序存储） ​

特点 ​

  1. 随机访问，能在O(1)时间内找到指定元素
  2. 存储密度高
  3. 拓展容量不方便
  4. 插入、删除数据不方便

顺序表的实现（静态分配） ​

静态分配的顺序表满了就无法继续添加了，因为其表长在开始确认后就无法更改（存储空间是静态的）

既大小一旦确定就无法改变

c
#include <stdio.h>
#define MaxSize 100 //定义最大长度
#define ElemType int //定义数组的类型
typedef struct{
    ElemType data[MaxSize]; //静态"数组"存放数据元素
    int length;//顺序表的当前长度
}SqList;//顺序表的类型定义（静态分配方式）

//基本操作--初始化一个顺序表
void InitList(SqList *L){
    L->length = 0; //顺序表初始长度为0
}
int main(){
    SqList L;
    InitList(&L);
}

顺序表的实现（动态分配） ​

在c语言中可以使用malloc和free动态申请内存和释放内存

顺序表满时可以再用malloc动态拓展最大容量

需要将元素复制到新申请的内存空间中，再用free函数释放原空间

c
#include <stdio.h>
#define InitSize 10 //定义初始大小
#define ElemType int//定义数据类型
typedef struct{
    ElemType *data;
    int MaxSize;
    int Length;
}SeqList;
//初始化线性表
void InitList(SeqList *L){
    //用malloc动态申请一段连续的内存空间
    L->data = (ElemType *)malloc(sizeof(ElemType) * InitSize);
    L->Length = 0;
    L->MaxSize = InitSize;
}
//malloc增加动态数组的长度
void IncreaseSize(SeqList *L,int len){
    ElemType *p = L->data;
    L.data = (ElemType *)malloc(sizeof(ElemType) * (InitSize+len));
    for(int i=0;i<L->length;i++){
        L->data[i] = p[i]; //复制数据到新区域
    }
    L->MaxSize = InitSize + len;//更新顺序表大小
    free(p);//释放原来空间
}
c
//使用realloc增加动态数组的长度
void IncreaseSize(SeqList *L,int len){
    L->MaxSize = InitSize + len;//更新顺序表大小
    ElemType *NewData = realloc(L->data,sizeof(ElemType) * L->MaxSize);
    L->data = NewData;
}

插入操作的时间复杂度

关注最深层循环语句的执行次数与问题规模n的关系

  • 最好情况

    新元素插入表尾，不需要移动元素，循环0次

    最好时间复杂度O(1)

  • 最坏情况

_Bool ListDelete(SeqList *L,int i){
    //判断位序是否合法
    if(i<0 || i - 1 >L->Length){
        return 0;
    }
    //将i位置后面的元素前移
    //注意位序、数组下标的关系，并从前面的元素开始依次移动
    for(int j=i;j<L->Length;j++){
        L->data[j-1] = L->data[j];
    }
    L->Length--;
    return 1;
}

时间复杂度

问题规模n=L->Length

最好情况 ：目标在表头，循环1次，最好时间复杂度O(1)

最坏情况：目标表表尾，循环n次，最坏时间复杂度O(n)

平均情况：

假设目标元素出现在每个位置的概率相同都是1n那么出现在第一位，循环1次；出现在第二位，循环2次...出现在n位，循环n次平均循环次数=11n+21n+...+n1n=n(n+1)21n=n+12假设目标元素出现在每个位置的概率相同都是\frac{1}{n}\\ 那么出现在第一位，循环1次；出现在第二位，循环2次...出现在n位，循环n次\\ 平均循环次数=1\frac{1}{n}+2\frac{1}{n}+...+n\frac{1}{n} = \frac{n(n+1)}{2}\frac{1}{n}=\frac{n+1}{2}假设目标元素出现在每个位置的概率相同都是n1​那么出现在第一位，循环1次；出现在第二位，循环2次...出现在n位，循环n次平均循环次数=1n1​+2n1​+...+nn1​=2n(n+1)​n1​=2n+1​

c
//在表list第i个位置插入元素e
_Bool ListInsert(LinkList L,int i,ElemType e){
    if(i<1) return 0;
    LNode *p;//指针p指向当前扫描到的节点
    int j = 0;//p指向的是第几个节点
    p = L;//L为头节点，为第0个节点，头节点不存放数据
    while(p!=NULL && j<i-1){//循环找i-1个节点(插入第i个位置，需要找前一个节点)
        p = p->next;
        j++;
    }
    if(p==NULL) return 0; //位置不合法(i位置节点为NULL,后续无法插入)
    LNode *s = (LNode *)malloc(sizeof(LNode));
    s->date = e;
    s->next = p->next; //对节点进行链接 (注意链接的先后顺序)
    p->next = s;
    return 1;
}

        s->data = e;
        s->next = L;
        L = s;//头指针指向新节点
        return 1;
    }
    LNode *p; //p指向当前扫描的节点
    int j = 1;//p指向第几个节点
    p = L;//（不是头结点）
    while(p!=NULL&&j<i-1){
        p=p->next;
        j++;
    }
    if(p == NULL) return 0; //i值是否合法
    LNode *s = (LNode *)malloc(sizeof(LNode));
    s->next = L->next;
    s->data = e;
    p->next = s;
    return 1;
}

//删除表L第i个元素，返回删除的元素值

    int j =0;
    p = L;
    while(p!=NULL&&j<i-1){
        p=p->next;

    free(q);//释放空间
    return 1;
}

LinkList List_TailInsert(LinkList L){ //正向建立单链表
    int x; //设置ElemType 为整形
    L = (LinkList)malloc(sizeof(LNode)); //建立头节点
    LNode *s,*r = L; //初始话指针，r为表尾指针

        s = (LNode *)malloc(sizeof(LNode));

        r->next = s;
        r = s;   //更新r指向表尾地址
        scanf("%d",&x);
    }
    r->next = NULL;//尾节点指向NULL
    return L;
}

静态链表 ​

什么是静态链表 ​

单链表：各节点在内存中随机分布

静态链表：分配一整片连续的内存空间，各节点安置其中（用数组的方式实现链表）

静态链表在内存中分配的空间

优点：增删操作不需要大量移动数据

缺点：不能随机存储，只能从头节点向后访问，容量大小固定不可变

适用场景：

  1. 不支持指针的低级语言
  2. 数据元素固定不变的场景
定义一个静态链表 ​
c
#define MaxSize 10 //静态链表的长度
struct Node{   //静态链表结构类型的定义
    ElemType e; //存储的数据元素
    int next; //下一个元素的数值下标
};
查找 ​

从头节点出发依次往后遍历

插入 ​

插入位序为i的节点

  1. 找到一个空节点，存入数据元素（初始化数组同时初始化数据，以进行空节点判断，不然会有脏数据）
  2. 从头节点出发，找到i-1位置的节点
  3. 修改新节点next
  4. 修改i-1节点的next
删除元素 ​

删除某个节点时

  1. 从头节点出发找到前驱节点
  2. 前驱节点更新为删除节点的next
  3. 被删除节点的next改为约定为空时的值

顺序表 Vs 链表 ​

存储结构 ​

都属于线性表，都是线性结构

顺序表：

  • 支持随机存储，存储密度高
  • 大片连续空间分配不方便，改变容量不方便

链表

  • 离散的小空间，分配容量方便，存储空间可变
  • 存储密度低，不能随机存储

基本操作 ​

创建、销毁、增删改查

顺序表：

  1. 创建->需要分配大片连续空间，不好掌控大小（动态分配【malloc、free】：动态数组，静态分配：静态数组）
  2. 销毁->静态：系统自动回收，动态->需要手动free
  3. 增、删->需要移动元素，时间复杂度O(n) 来自移动元素（若元素很大，代价较高)
  4. 按位查找->O(1);按值查找->O(n)

链表：

  1. 创建-> 只需要声明头节点或头指针，方便后续扩展
  2. 销毁->依次删除每个节点
  3. 增、删->需要修改指针，时间复杂度O(n) 来自查找元素(查找元素的时间代价更低)
  4. 按位查找->O(n);按值查找->O(n)
